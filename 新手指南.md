## 1. 项目是什么？

这是一个基于 **Jekyll** 框架的静态网站，由 GitHub Pages 免费托管。

### 工作原理
```
你编辑 Markdown/YAML 文件
        ↓
推送到 GitHub
        ↓
GitHub Pages 自动构建
        ↓
网站更新（1-2分钟后）
```

### 网站地址
- **你的网站**: https://zhiqinzhou.github.io
- **仓库地址**: https://github.com/ZhiqinZhou/ZhiqinZhou.github.io

## 2. 目录结构详解

### 🔴 你需要关注的（可编辑）

| 文件/文件夹 | 作用 | 说明 |
|-------------|------|------|
| `_config.yml` | **核心配置** | 网站标题、你的个人信息、社交链接 |
| `_pages/about.md` | **主页内容** | 访客看到的第一个页面 |
| `_pages/cv.md` | **简历页面** | 你的简历/履历 |
| `_data/navigation.yml` | **导航栏** | 控制顶部显示哪些菜单 |
| `images/` | **图片文件夹** | 放头像、项目截图等 |
| `files/` | **下载文件夹** | 放PDF简历等可下载文件 |

### 🟡 内容模块（按需使用，目前为空）

| 文件夹 | 作用 | 何时使用 |
|--------|------|----------|
| `_publications/` | 论文发表 | 有学术论文时 |
| `_portfolio/` | 作品集 | 展示项目作品时 |
| `_posts/` | 博客文章 | 想写博客时 |
| `_talks/` | 演讲报告 | 有演讲经历时 |
| `_teaching/` | 教学内容 | 有教学经历时 |

### 🟢 系统文件（不要动！）

| 文件/文件夹 | 作用 | 为什么不能动 |
|-------------|------|-------------|
| `_includes/` | HTML模板片段 | 网站构建需要 |
| `_layouts/` | 页面布局模板 | 网站构建需要 |
| `_sass/` | CSS样式文件 | 网站外观需要 |
| `assets/` | JS/CSS资源 | 网站功能需要 |
| `.git/` | Git版本控制 | 代码管理需要 |
| `Gemfile` | Jekyll依赖 | 构建需要 |

## 3. 你需要编辑的文件

### 3.1 个人信息配置 `_config.yml`

这是最重要的文件，打开后找到 `author:` 部分（约第23行）：

```yaml
author:
  avatar: "profile.png"      # 头像文件名（放在 images/ 文件夹）
  name: "Zhiqin Zhou"        # 你的名字
  bio: "Welcome to my site"  # 一句话简介，显示在侧边栏
  location: "China"          # 你的位置
  email: "your@email.com"    # 你的邮箱
  github: "ZhiqinZhou"       # GitHub用户名
  linkedin: "yourname"       # LinkedIn用户名（可选）
  weibo: "yourname"          # 微博用户名（可选）
```

**可用的社交链接**（填入用户名即可显示图标）：
- 学术：`googlescholar`, `orcid`, `researchgate`
- 社交：`twitter`, `linkedin`, `weibo`, `zhihu`, `instagram`
- 代码：`github`, `kaggle`, `stackoverflow`

### 3.2 主页内容 `_pages/about.md`

这是访客看到的第一个页面。用 Markdown 语法编写：

```markdown
---
permalink: /
title: "Welcome"
author_profile: true
---

# 关于我

我是XXX，目前就读于XXX大学XXX专业。

## 研究兴趣

- 机器学习
- 自然语言处理

## 联系方式

欢迎通过左侧链接联系我！
```

### 3.3 简历页面 `_pages/cv.md`

当前包含示例内容，你需要替换成自己的简历：

```markdown
---
layout: archive
title: "CV"
permalink: /cv/
author_profile: true
---

## 教育经历

- 2020-2024，XX大学，XX专业，本科

## 技能

- Python, PyTorch
- 机器学习, 深度学习
```

### 3.4 导航栏配置 `_data/navigation.yml`

控制网站顶部显示哪些菜单。取消注释（删除 `#`）即可显示：

```yaml
main:
  # 取消下面的注释来显示对应菜单
  # - title: "Publications"
  #   url: /publications/
  
  - title: "CV"           # 当前显示
    url: /cv/
    
  # - title: "Blog Posts"  # 取消注释来启用博客
  #   url: /year-archive/
```

## 4. 常见操作教程

### 4.1 添加头像

1. 准备一张头像图片
2. 重命名为 `profile.png`
3. 放入 `images/` 文件夹
4. 确保 `_config.yml` 中 `avatar: "profile.png"`

### 4.2 添加一篇博客

1. 在 `_posts/` 文件夹创建文件，命名格式：`YYYY-MM-DD-文章标题.md`
   
   例如：`2024-01-17-my-first-post.md`

2. 文件内容格式：
```markdown
---
title: '我的第一篇博客'
date: 2024-01-17
permalink: /posts/2024/01/my-first-post/
tags:
  - 学习笔记
---

这里是文章正文...
```

3. 在 `_data/navigation.yml` 中启用博客菜单（取消注释）

### 4.3 添加项目作品

1. 在 `_portfolio/` 文件夹创建 `project-1.md`：

```markdown
---
title: "项目名称"
excerpt: "项目简介"
collection: portfolio
---

项目详细介绍...
```

2. 在导航栏启用 Portfolio 菜单

### 4.4 上传PDF简历

1. 将 `CV.pdf` 放入 `files/` 文件夹
2. 在任意页面添加下载链接：
```markdown
[下载我的简历](../files/CV.pdf)
```

## 5. 发布更新到网站

每次修改文件后，执行以下命令推送到 GitHub：

```powershell
# 在项目目录下打开终端，执行：
git add .
git commit -m "更新网站内容"
git push
```

等待 1-2 分钟，访问 https://zhiqinzhou.github.io 查看更新。

## ❓ 常见问题

**Q: 修改后网站没变化？**
- GitHub Pages 构建需要 1-2 分钟，请耐心等待
- 清除浏览器缓存后刷新

**Q: 如何本地预览？**
- 需要安装 Ruby + Jekyll 环境（较复杂）
- 建议直接推送到 GitHub 预览

**Q: 可以换主题颜色吗？**
- 修改 `_config.yml` 第11行 `site_theme`
- 可选：`default`, `air`, `sunrise`, `mint`, `dirt`, `contrast`
